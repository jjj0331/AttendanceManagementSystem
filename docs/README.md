# プロジェクトドキュメント

## 概要

AI駆動開発のためのドキュメントテンプレートです。

---

## ドキュメント構成

```
docs/
├── 00_ガイド/             # 開発の「やり方」を説明
├── 10_アーキテクチャ/     # 技術的な「何」を定義（初期決定）
├── 20_ドメイン/           # ビジネスの「何」を表現
├── 30_設計/               # 技術的な「どうやって」を表現
├── 40_決定記録/           # 決定の「なぜ」を記録（変更履歴）
├── 50_計画/               # プロジェクトの「いつ」を管理
├── 60_テスト/             # テスト計画・結果
├── 70_運用/               # 本番運用の「どう維持する」を記録
└── 99_アーカイブ/         # 過去の記録
```

## AI駆動開発スキル

このテンプレートで使用できるスキル：

| Phase | スキル | 用途 | 出力先 |
|-------|--------|------|--------|
| 0 | `/project-overview` | プロジェクト概要生成 | `20_ドメイン/プロジェクト概要.md` |
| 0.5 | `/discovery-session` | ドメイン知識抽出 | `20_ドメイン/ディスカバリー/` |
| 1 | `/event-storming` | ES図生成 | `20_ドメイン/イベントストーミング/{context}.md` |
| 2 | `/domain-modeling` | ドメインモデル生成 | `20_ドメイン/{context}/{aggregate}集約.md` |
| 3 | `/api-design` | API設計生成 | `30_設計/API/エンドポイント/{aggregate}.md` |
| 3 | `/db-design` | DB設計生成 | `30_設計/データベース/{aggregate}.md` |
| 3 | `/screen-design` | 画面設計生成 | `30_設計/画面/screens/{screen}.md` |
| 3.5 | `/cross-context-design` | コンテキスト間連携設計 | `30_設計/コンテキスト間連携.md` |
| - | `/design-review` | 設計実装可能性レビュー | `tmp/design-review-report.md` |
| - | `/estimate` | 工数見積もり自動生成 | `80_見積もり/{project}_見積もり_{YYYYMMDD}.md` |
| 4 | `/scaffold` | スキャフォールド生成 | プロジェクトルート |
| - | `/create-backlog-issue` | Backlog課題一括生成 | Backlog |
| - | `/sync` | Miro/Markdown整合性チェック | - |

### オプション

| スキル | オプション | 説明 |
|--------|-----------|------|
| `/discovery-session` | `--quick` | 30分1セッション版 |
| `/discovery-session` | `--session {N}` | Session Nから再開 |
| `/event-storming` | `--from-discovery` | ES用サマリーから生成 |
| `/event-storming` | `--sync` | Miroから同期 |
| `/domain-modeling` | `--minimal` | Must質問12問のみ |
| `/domain-modeling` | `--sync` | ES図/Miroから同期 |
| `/api-design` | `--format openapi` | OpenAPI Spec形式 |
| `/db-design` | `--format sql` | DDL形式 |

詳細: [AI駆動開発ワークフロー](./00_ガイド/AI駆動開発/ワークフロー.md)

## フォルダ説明

### 00_ガイド/

開発者向けのガイドドキュメントを格納します。

### 10_アーキテクチャ/

システムの技術的なアーキテクチャを定義します。

### 20_ドメイン/

ビジネスドメインの知識を格納します。

```
20_ドメイン/
├── プロジェクト概要.md      # 企画書（Phase 0）
├── ディスカバリー/          # ドメイン知識抽出（Phase 0.5）
├── イベントストーミング/    # ES図（Phase 1）
├── {コンテキスト名}/        # 集約ドキュメント（Phase 2）
├── コンテキストマップ.md    # コンテキスト間の関係
├── 用語集.md               # ユビキタス言語
└── 業務ルール/             # ビジネスルール
```

### 30_設計/

技術的な詳細設計ドキュメントを格納します（Phase 3 の出力先）。

```
30_設計/
├── API/                    # API仕様（/api-design）
├── データベース/            # DB設計（/db-design）
└── 画面/                   # UI設計（/screen-design）
```

### 40_決定記録/

重要な技術的決定とその理由を記録します。

### 50_計画/

実装計画とロードマップを格納します。

### 60_テスト/

テスト計画と結果を格納します。

### 70_運用/

本番運用に関するドキュメントを格納します。

```
70_運用/
├── デプロイ/              # デプロイ手順・設定
├── 監視/                  # 監視・アラート設定
├── インシデント対応/       # 障害対応手順
└── ランブック/            # 定型運用作業
```

### 99_アーカイブ/

過去の記録・参考資料を格納します。
