# AI駆動開発 ガイド

## 概要

AIと協調して効率的にソフトウェア開発を進めるための方法論を格納しています。

---

## 背景

### 従来の課題

| 課題 | 説明 |
|------|------|
| 設計とコードの乖離 | ドキュメントが古くなり、実装と一致しなくなる |
| AIへの文脈伝達 | 設計意図をAIに正確に伝えるのが難しい |
| 知識の属人化 | ビジネスルールが実装者の頭の中にしかない |

### 目指す姿

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   設計資料（Miro + Markdown）  ←──→  AI（Claude Code）     │
│            ↓                            ↓                  │
│   Source of Truth                  自動コード生成          │
│   （常に最新を維持）                （設計に準拠）          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 核となるコンセプト

### 1. 二層構造の設計資料

```
Miro（イベントストーミング図）
  └── What: 何があるか（集約、コマンド、イベント、リードモデル）

Markdown + Mermaid（ドメインモデリング）
  └── How: どう動くか
      ├── 構造図（クラス図）
      ├── 状態遷移図
      ├── コマンド・イベント定義
      ├── クエリ定義
      └── 不変条件
```

### 2. 対話による知識蓄積

```
初回: AIが質問 → 人間が回答 → Markdownに記録
次回: Markdownを参照 → 既知のルールは自動適用
```

### 3. 段階的な詳細化

```
Phase 0: プロジェクト概要     → 基本情報を整理
Phase 1: イベントストーミング → 大枠を決定（Miro）
Phase 2: ドメインモデリング   → 構造・振る舞いを定義（Markdown）
Phase 3: 詳細設計            → API・DB・画面設計
Phase 4: 実装                → コード生成
```
