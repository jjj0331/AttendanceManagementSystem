## 2. API→画面マッピング

### 休暇申請一覧

| API | メソッド | 画面での用途 |
|-----|---------|-------------|
| `/leave-requests` | GET | 一覧表示（検索・ソート・ページネーション） |

### 休暇申請詳細

| API | メソッド | 画面での用途 |
|-----|---------|-------------|
| `/leave-requests/{id}` | GET | 詳細表示 |
| `/leave-requests/{id}/actions/approve` | POST | 承認ボタン（上長） |
| `/leave-requests/{id}/actions/reject` | POST | 却下ボタン（上長） |
| `/leave-requests/{id}/actions/cancel` | POST | 取り消しボタン（申請者） |

### 休暇申請作成

| API | メソッド | 画面での用途 |
|-----|---------|-------------|
| `/leave-requests` | POST | 休暇申請の新規作成 |

### 承認待ち休暇申請一覧

| API | メソッド | 画面での用途 |
|-----|---------|-------------|
| `/leave-requests/pending-approvals` | GET | 承認待ち一覧表示（検索・ソート・ページネーション） |

## 6. 推奨実装順序

| 順序 | 画面名 | 理由 |
|------|--------|------|
| 1 | 休暇申請作成 | コア操作。従業員が休暇申請を行う起点。休暇種別による動的フォーム切替が要 |
| 2 | 休暇申請一覧 | 申請者が自分の申請状況を確認する基本画面 |
| 3 | 休暇申請詳細 | 承認・却下・取り消しのアクションを集約。一覧からの遷移先 |
| 4 | 承認待ち休暇申請一覧 | 上長向け。詳細画面完成後に対で使用 |
