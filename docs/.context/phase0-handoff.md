# Phase 0 → Phase 0.5 申し送り

## 完了サマリー

勤怠管理システム（Kintai）のプロジェクト概要書と技術スタックを策定。従業員500名規模の社内向けWebシステムで、打刻・残業・休暇管理を中心としたMVPスコープを定義した。

## 次のPhaseへの入力

- **プロジェクト種別**: 社内業務システム（勤怠管理）
- **主要ドメイン**: 勤怠打刻、残業管理、休暇管理、シフト管理、月次締め処理
- **技術制約サマリー**: monolith / standard（軽量CQRS） / REST / JWT + Google SSO
- **ディスカバリー推奨モード**: 通常モード（`/discovery-session`）
  - 理由: 労働基準法・36協定・有給付与ルールなど、法令起因の業務ルールが多く深掘りが必要
- **architecture_pattern**: `standard`

## 未確定事項

- [ ] 既存給与システムとの連携仕様（理由: 給与システム側のAPI/ファイル仕様が未提供）
- [ ] 契約社員・パートタイマーの勤務パターン対応範囲（理由: MVP外だが影響範囲の見積もりが必要）
- [ ] リモートワーク打刻の位置確認ポリシー（理由: GPS取得の法的整理が必要）

## 注意点

- 労働基準法（36協定）の上限値は法改正で変わりうる。ハードコードせず設定値として外部化すること
- 有給休暇の付与ルールは入社日起算で複雑（6ヶ月後10日、1.5年後11日...）。テーブルドリブンで管理推奨
- 「打刻修正」は承認ワークフローが必須。無承認修正は監査上のリスク
- 夜勤（日跨ぎ）の勤務時間計算は要注意。暦日ではなく勤務日ベースの判定が必要
